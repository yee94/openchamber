# Actions de projet

Une action de projet est une commande shell que vous enregistrez une fois et lancez d’un clic — votre serveur de dev, un build, une passe de tests. Chaque projet garde sa propre liste. Configurez-les dans **Paramètres → Projets → Actions de projet**.

## Ajouter une action

1. Ouvrez **Paramètres → Projets** et trouvez la section **Actions de projet**.
2. Ajoutez une action, donnez-lui un nom, choisissez une icône et saisissez la commande à lancer.
3. Enregistrez.

Vous pouvez limiter une action à certains systèmes d’exploitation si une commande n’a de sens que sur l’un d’eux.

## Lancer une action

Les actions vivent dans un menu dans l’en-tête de l’application. Cliquez sur l’une d’elles et OpenChamber la lance dans un terminal dans le dossier de votre projet, puis bascule vers la vue terminal pour que vous puissiez suivre la sortie. Arrêtez-la depuis le même menu.

## Ouvrir automatiquement un serveur de dev

Activez **auto-open URL** pour une action qui démarre un serveur. OpenChamber surveille la sortie pour détecter une adresse locale et propose de l’ouvrir — voir [Aperçu et serveurs de dev](/preview/). Sur desktop, vous pouvez aussi la faire passer par un transfert de port SSH.
