# Tunnels desktop

L’application desktop peut créer un tunnel public depuis **Paramètres → OpenChamber → Tunnel**. Vous n’avez pas besoin de démarrer OpenChamber depuis la CLI pour ce chemin.

## Installer un fournisseur

OpenChamber démarre la CLI du fournisseur sur votre machine. Installez d’abord le fournisseur que vous voulez utiliser :

```bash
brew install cloudflared
brew install ngrok
```

Cloudflare utilise `cloudflared`. Ngrok nécessite un compte ngrok et un authtoken depuis le dashboard ngrok :

```bash
ngrok config add-authtoken <your-ngrok-token>
```

## Démarrer depuis l’application

1. Ouvrez **Paramètres → OpenChamber → Tunnel**.
2. Choisissez **Cloudflare** ou **Ngrok**.
3. Démarrez un tunnel rapide.
4. Scannez le QR code généré depuis votre téléphone.

Ngrok prend actuellement en charge les tunnels rapides. Cloudflare prend en charge les tunnels rapides et les modes Cloudflare gérés.

## Protection de l’accès

Même lorsque l’URL du fournisseur est publique, OpenChamber protège toujours l’accès avec son propre token de connexion. Le lien de connexion généré contient un token à usage unique, possède une TTL, et les anciens liens inutilisés sont révoqués lorsque vous en générez un nouveau ou que vous arrêtez/redémarrez le tunnel.
