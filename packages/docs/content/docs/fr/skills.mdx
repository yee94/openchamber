# Skills

Un skill est un ensemble réutilisable d’instructions qu’un agent peut charger quand c’est pertinent — par exemple « comment nous écrivons les messages de commit » ou « nos conventions d’API ». Gérez-les dans **Paramètres → Skills**.

Pour installer des skills prêts à l’emploi au lieu d’écrire les vôtres, voir [Catalogue de skills](/skills-catalog/).

## Créer un skill

1. Ouvrez **Paramètres → Skills**.
2. Créez un skill et donnez-lui un nom et une courte description. La description est ce qui permet à l’agent de décider quand le skill s’applique, donc soyez précis.
3. Écrivez les instructions. Ajoutez des fichiers de support si le skill en a besoin.
4. Choisissez où il vit :
   - **personal** — disponible dans tous les projets
   - **project** — disponible uniquement dans le projet actuel

## Utiliser un skill dans le chat

Tapez `/` au milieu d’un message pour afficher le sélecteur de skills, puis choisissez-en un. L’agent charge les instructions de ce skill pour la réponse.

Un `/` au tout début du message ouvre plutôt les [commandes](/commands-snippets/).
