# Projets

Un projet est un dossier de votre ordinateur qu’OpenChamber suit — généralement un codebase. Changer de projet change le dossier dans lequel l’agent travaille, ainsi que les sessions et paramètres de ce projet.

## Ajouter un projet

Vous pouvez ajouter un projet depuis plusieurs endroits :

- l’entrée **Ajouter un projet** dans la palette de commandes
- le bouton **+** en haut de la barre latérale des sessions
- le navigateur de dossiers lorsque vous choisissez un répertoire

Pointez vers un dossier et OpenChamber s’en souvient. Le nom vient du dossier ; vous pouvez le modifier plus tard.

## Changer de projet

Choisissez un projet dans la barre latérale pour le rendre actif. Tout — sessions, git, notes — suit le projet ouvert.

## Rendre un projet reconnaissable

Ouvrez **Paramètres → Projets** pour donner à un projet un nom, une couleur ou une icône personnalisés. OpenChamber essaie de trouver une icône automatiquement — voir [Icônes de projet](/project-icons/).

> Dans VS Code, OpenChamber utilise toujours le dossier ouvert comme seul projet ; il n’y a donc rien à ajouter ni à changer. La page de paramètres Projets y est masquée.
