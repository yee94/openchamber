# Notifications

Les notifications vous signalent quand quelque chose demande votre attention, sans devoir surveiller l’écran — une session s’est terminée, a rencontré une erreur, a posé une question ou a besoin d’une permission. Configurez-les dans **Paramètres → OpenChamber → Notifications**.

## Les activer

1. Ouvrez **Paramètres → OpenChamber → Notifications**.
2. Autorisez les notifications lorsque votre navigateur ou système le demande.
3. Choisissez ce dont vous voulez être informé :
   - une session **se termine**
   - une session rencontre une **erreur**
   - une session **pose une question**
   - une session a besoin d’une **permission**
   - des **sous-tâches** se terminent

## Comment elles vous atteignent

- sur **desktop**, vous recevez des notifications système natives
- dans un **navigateur ou une application installée**, vous recevez des notifications web push, qui arrivent même lorsque l’onglet est en arrière-plan

Les sessions configurées en auto-accept ne vous dérangent pas avec des notifications de permission.

## Personnaliser le texte

Chaque type de notification a un modèle de titre et de message que vous pouvez modifier, avec des champs comme le nom de l’agent et le modèle. Il existe aussi une limite sur la quantité du dernier message incluse, pour que les notifications restent courtes.
